Triển khai Domain Controller + DNS + AD CS trên Windows Server
----

# 1. Chuẩn bị
- Đặt IP tĩnh.
- Đổi tên Server.
- Đặt DNS trỏ về chính máy.
- Đổi hostname, restart.

# 2. Cài Active Directory Domain Services (Đã hoàn thành từ bài trước)
```
Server Manager
→ Add Roles and Features
→ Active Directory Domain Services
→ Install
```

Promote Domain Controller
```
Add a new Forest

Domain:
seta-international.vn
```
Restart.

Kiểm tra:
```
Server Manager → Tools

✓ Active Directory Users and Computers
✓ DNS
✓ Group Policy Management

```

# 3. DNS (Đã hoàn thành từ bài trước)

Sau khi Promote sẽ tự sinh
```
Forward Lookup Zones

_msdcs.seta-international.vn
seta-international.vn

```

## 3.1 Forward Lookup Zone (Thường dùng)

Chức năng
```
Tên máy

↓

Địa chỉ IP
```

Ví dụ
```
dc01

↓

192.168.90.10
```

Dùng khi

- Thêm Server
- Thêm NAS
- IIS
- File Server
- ERP

Tạo bản ghi
```
Forward Lookup Zone

↓

New Host (A)

↓

Tên

↓

IP
```

## 3.2 Reverse Lookup Zone

Chức năng
```
IP

↓

Tên máy
```

Ví dụ
```
192.168.90.10

↓

dc01.seta-international.vn
```
Dùng khi

- Kiểm tra log
- Mail Server
- SIEM
- Audit
- nslookup

Tạo
```
Reverse Lookup Zones

↓

New Zone

↓

IPv4

↓

Network ID

192.168.90
```
Sau đó tạo PTR Record.

## 3.3 Conditional Forwarder

Không lưu bản ghi DNS.

Chỉ dùng để chuyển truy vấn DNS sang DNS Server khác.

Ví dụ
```
Domain A

↓

seta-international.vn
```
Muốn phân giải
```
blueoc.vn
```
Tạo
```
Conditional Forwarder

blueoc.vn

↓

192.168.100.10
```

Khi nào dùng

- Hai Domain
- Hai Site
- Site-to-Site VPN
- Forest khác
- Azure
- AWS

Không dùng để tạo Host A.

# 4. Tạo Subdomain

Có 2 cách.

**Cách 1**

Nếu Subdomain dùng cùng DNS
```
Forward Lookup Zones

↓

New Zone

↓

Primary Zone

↓

lab.seta-international.vn
```
Ví dụ
```
lab.seta-international.vn
dev.seta-international.vn
test.seta-international.vn
```

**Cách 2**

Nếu Subdomain có DNS Server riêng
```
New Delegation
```

Ví dụ
```
dev.seta-international.vn

↓

DNS Server khác quản lý
```

Khuyến nghị

```
Cùng Server → New Zone
Khác Server → Delegation
```

# 5. Cài Active Directory Certificate Services (AD CS)

**Kiểm tra**

PowerShell với quyền Administrator
```
Get-WindowsFeature ADCS-Cert-Authority
```
Nếu
```
Available
```

→ Chưa cài.

Nếu

```
Installed
```
→ Đã cài.

**Cài đặt**

```
Server Manager

↓

Add Roles and Features

↓

Active Directory Certificate Services

↓

Certification Authority
```

Install.

**Configure**

```
Enterprise CA

↓

Root CA

↓

Create New Private Key

↓

RSA 4096

↓

SHA256
```

Finish.

# 6. Export Root Certificate

```
Certification Authority

↓

Properties

↓

View Certificate

↓

Details

↓

Copy To File
```

Chọn

```
DER (.cer)
```
Lưu
```
RootCA.cer
```
