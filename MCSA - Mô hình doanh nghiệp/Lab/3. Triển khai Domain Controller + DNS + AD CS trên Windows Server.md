Triển khai Domain Controller + DNS + AD CS trên Windows Server
----

# 1. Chuẩn bị
- Đặt IP tĩnh.
- Đổi tên Server.
- Đặt DNS trỏ về chính máy.
- Đổi hostname, restart.

# 2. Cài Active Directory Domain Services (Đã hoàn thành từ bài trước)
```
Server Manager
→ Add Roles and Features
→ Active Directory Domain Services
→ Install
```

Promote Domain Controller

```
Add a new Forest

Domain:
seta-international.vn
```
Restart.

Kiểm tra:

```
Server Manager → Tools

✓ Active Directory Users and Computers
✓ DNS
✓ Group Policy Management

```

# 3. DNS (Đã hoàn thành từ bài trước)

Sau khi Promote sẽ tự sinh
```
Forward Lookup Zones

_msdcs.seta-international.vn
seta-international.vn

```

## 3.1 Forward Lookup Zone (Thường dùng)

Chức năng
```
Tên máy

↓

Địa chỉ IP
```

Ví dụ
```
dc01

↓

192.168.90.10
```

Dùng khi

- Thêm Server
- Thêm NAS
- IIS
- File Server
- ERP

Tạo bản ghi

```
Forward Lookup Zone

↓

New Host (A)

↓

Tên

↓

IP
```

## 3.2 Reverse Lookup Zone

Chức năng
```
IP

↓

Tên máy
```

Ví dụ
```
192.168.90.10

↓

dc01.seta-international.vn
```
Dùng khi

- Kiểm tra log
- Mail Server
- SIEM
- Audit
- nslookup

Tạo
```
Reverse Lookup Zones

↓

New Zone

↓

IPv4

↓

Network ID

192.168.90
```
Sau đó tạo PTR Record.

## 3.3 Conditional Forwarder

Không lưu bản ghi DNS.

Chỉ dùng để chuyển truy vấn DNS sang DNS Server khác.

Ví dụ
```
Domain A

↓

seta-international.vn
```
Muốn phân giải
```
blueoc.vn
```
Tạo
```
Conditional Forwarder

blueoc.vn

↓

192.168.100.10
```

Khi nào dùng

- Hai Domain
- Hai Site
- Site-to-Site VPN
- Forest khác
- Azure
- AWS

Không dùng để tạo Host A.

# 4. Tạo Subdomain

Có 2 cách.

**Cách 1**

Nếu Subdomain dùng cùng DNS

```
Forward Lookup Zones

↓

New Zone

↓

Primary Zone

↓

lab.seta-international.vn
```
Ví dụ
```
lab.seta-international.vn
dev.seta-international.vn
test.seta-international.vn
```

**Cách 2**

Nếu Subdomain có DNS Server riêng
```
New Delegation
```

Ví dụ
```
dev.seta-international.vn

↓

DNS Server khác quản lý
```

Khuyến nghị

```
Cùng Server → New Zone
Khác Server → Delegation
```

Sau khi cài xong sẽ vào cửa sổ cấu hình

**1/ Credentials: Kiểm tra xem đang để User hay Administrator**

<img width="765" height="557" alt="image" src="https://github.com/user-attachments/assets/572f5aca-b50e-46ce-9f3e-7196f17bc0f4" />

-> Next

**2/ Role Services**

Chỉ tích chọn **Certification Authority**

-> Next

**3/ Setup Type**

Chọn: **Enterprite CA**

**Giải thích:**
- **Enterprite CA**: Tích hợp với Active Directory, tự động cấp phát và quản lý chứng chỉ cho máy tính, người dùng trong Domain. Đây là lựa chọn phù hợp cho hầu hết doanh nghiệp và môi trường lab có Domain.
- **Standalone CA**: Không tích hợp AD, phải cấp phát thủ công. Thường dùng cho Root CA offline hoặc các trường hợp đặc biệt

-> Next

**4/ CA Type**

Chọn: **Root CA**

**Giải thích**:
- **Root CA**: CA gốc, tự ký chứng chỉ của chính nó và là gốc tin cậy của toàn bộ hệ thống PKI
- **Subordinate CA**: CA cấp dưới, nhận chứng chỉ từ Root CA và dùng để cấp chứng chỉ cho người dùng/dịch vụ.

-> Next

**5/ Private Key**

Chọn: **Create a new private key**

-> Next

**6/ Cryptography**

- **Hash Algorithm**: SHA256

# 6. Export Root CA Certificate (.cer)

**1/ Mở Certification Authority**

Vào:
```
Server Manager
→ Tools
→ Certification Authority
```

Hoặc nhấn:

```
certsrv.msc
```

**2/ Mở Properties của CA**

Trong cửa sổ Certification Authority:

- Chọn tên CA (ví dụ: SETA-ROOT-CA)
- Chuột phải → Properties

**3/ Xem Certificate**

Chọn tab:
```
General
```

Nhấn:
```
View Certificate
```

**4/ Export Certificate**

```
Chọn tab: Details

↓

Nhấn:

Copy to File...

Certificate Export Wizard xuất hiện.

↓

Nhấn:

Next
```

**5/ Chọn định dạng**

Chọn:

```
DER encoded binary X.509 (.CER)
```

Đây là định dạng chuẩn để triển khai qua Group Policy.

↓

Nhấn: Next

**6/ Đặt tên CA & Chọn nơi lưu**

Ví dụ:

```
C:\Temp\RootCA.cer
```
hoặc
```
D:\Certificate\RootCA.cer
```
↓

Nhấn: Next

↓

Finish

**7/ Kiểm tra**

Mở thư mục đã lưu, phải thấy file:
```
RootCA.cer
```

**Đây là Root Certificate (chỉ chứa khóa công khai), dùng để:**

- Import vào Group Policy.
- Import thủ công trên máy ngoài Domain.
- Phân phối cho các thiết bị như firewall, switch, NAS... để tin tưởng CA.

> Lưu ý: Không export **Private Key (.PFX)** ở bước này. Chỉ export file **.CER** vì mục đích là để các máy khách tin tưởng **CA**, không phải sao lưu khóa bí mật.

# 7. Deploy Root CA bằng Group Policy

## 7.1: Mở Domain

Bên trái mở theo thứ tự:

```
Forest: seta-international.vn
    └── Domains
            └── seta-international.vn
```
Sẽ thấy:

```
Domain Controllers
Group Policy Objects
Default Domain Policy
Default Domain Controllers Policy
```

<img width="759" height="534" alt="image" src="https://github.com/user-attachments/assets/3fb801d6-6054-46b2-8015-8fa13a65bbb2" />

## 7.2: Tạo GPO mới

Chuột phải vào:

```
seta-international.vn
```

Chọn:

```
Create a GPO in this domain, and Link it here...
```

Đặt tên:

```
Deploy Root CA Certificate
```

Nhấn OK.

> Lưu ý: Chọn **Create a GPO in this domain, and Link it here...** để GPO được tạo và tự động liên kết với toàn bộ Domain.

## 7.3: Chỉnh sửa GPO

Chuột phải vào GPO vừa tạo:

```
Deploy Root CA Certificate
```
↓

Chọn:

Edit

## 7.4: Import Root CA Certificate

Đi theo đường dẫn:

```
Computer Configuration
    └── Policies
        └── Windows Settings
            └── Security Settings
                └── Public Key Policies
                    └── Trusted Root Certification Authorities
```
Chuột phải:

Trusted Root Certification Authorities

↓

Chọn:

Import...


## 7.5: Import file RootCA.cer

Certificate Import Wizard xuất hiện.

- Next
- Browse
- Chọn file:

```
RootCA.cer
```

- Next
- Next
- Finish

Nếu hiện:

```
The import was successful.
```

→ Nhấn OK.

## 7.6: Áp dụng Group Policy

Trên máy Client chạy:
```
gpupdate /force
```
Hoặc khởi động lại máy.

## 7.7: Kiểm tra

Trên máy Client:

1. Nhấn Win + R
2. Gõ:
```
mmc
```
3. File → Add/Remove Snap-in
4. Chọn Certificates
5. Chọn Computer account

Mở:
```
Certificates (Local Computer)
    └── Trusted Root Certification Authorities
        └── Certificates
```

Phải nhìn thấy chứng chỉ của CA (ví dụ: **SETA-ROOT-CA**).
